## 2. 依存方針

core必須依存は最小化する。

推奨:

- Python 3.11以上
- 標準ライブラリ中心
- `typing_extensions`は必要な場合のみ

任意依存:

- NumPy
- Graphviz
- Pydantic
- Rich
- pytest
- Hypothesis

Chronowire自体は任意のPython値を流せる設計とし、NumPyをcore必須にしない案が望ましい。

## 4. Phase 1: 最小実行系

実装:

- PythonCallableKernel
- finite iterable Source
- 単一入力map
- DAG
- 分岐
- 複数終端
- ancestor抽出
- 単一スレッドScheduler
- FIFO buffer
- run()

完了例:

```python
base = cw.Flow([1, 2, 3], config)
a = base.map(lambda x: x + 1)
b = base.map(lambda x: x * 2)

run_result = cw.compile([
    cw.output(a, collector=cw.Bounded(max_items=3)),
    cw.output(b, collector=cw.Bounded(max_items=3)),
]).run()
ra, rb = run_result.outputs
```

期待:

```text
[item.value for item in ra.emissions] == [2, 3, 4]
[item.value for item in rb.emissions] == [2, 4, 6]
```

共通Sourceは一度だけ読み出す。

## 9. 公開version

必須機能、非対象、release gateの正本は[12_v0.1_v0.2リリース方針.md](12_v0.1_v0.2リリース方針.md)とする。以下はPhaseとの対応を示す要約である。

### v0.1

- Flow
- Config scope
- map
- frame
- compile(list)
- run
- NoCollect/Latest/Bounded collector
- Emission statusとDiagnostic
- Snapshot Extension
- finite/generated Source
- exact mergeとlatest state
- 共有PortBufferとbounded buffer planning
- realtime push Sourceとingress overflow
- multiple outputs
- graph_info
- JSON/DOT export
- Python backend
- 完全なPortablePlanIR descriptorとround-trip
- demand-driven Schedulerとproducer frontier
- realtime gap伝播

v0.1はPhase 0からPhase 3までの意味論上の必須項目を完了する。profiler、Mermaid/SVG、汎用的な最適化はPhase 3に記載されていてもv0.1 release gateには含めない。

### v0.2

- 包含、overlap、tolerance付き同期
- PlanSessionによる継続state
- multiple output Port
- StateFlow Source helperとRuntimeOptions
- PortablePlanIRの明示binding
- runtime profilerと長時間streaming評価

### v0.3

- `executor.py`のExecutor protocolとPythonExecutor
- sample/block CBFの基準traceと実行粒度測定
- PortablePlanIR schema 0.3のStage、value schema、実験的Kernel ABI
- SOURCE、RATE、FRAME、単一MAPのCython Executor
- PortablePlanIRとnative bufferのsemantic conformance
- Python callbackのStage境界

### v0.4

- version付きKernel ABI
- C++ ExecutorとPython binding
- Python/C++ Backend交換
- streaming CBFのC++ Stage実行

### v0.5

- block粒度、Fusion、Batch、parallel Stageの個別評価
- Executor間の性能・意味論比較
- representative evaluationと再現可能なbenchmark

### v0.9

- 実利用によるAPI usability review
- 0.x migration note
- 公開API、Diagnostic code、PortablePlanIR schemaのfreeze候補
- v1.0 tutorialとKernel authoring文書

### v1.0

- Python Executorの実行意味論安定
- C++ ExecutorとBackend交換
- API usability reviewと互換方針
- representative example、evaluation、benchmark
- 十分な試験、tutorial、Kernel authoring文書
- 最初のPyPI正式公開

0.xはGitHub tagで配布し、破壊的API変更を許容する。v0.3からv0.9の詳細な到達点とv1.0 release gateは[12_v0.1_v0.2リリース方針.md](12_v0.1_v0.2リリース方針.md)に定める。
